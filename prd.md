# PRD — 자장(Jajang)

**버전**: v1.1  
**작성일**: 2026-04-24  
**상태**: 확정 (Option B — Selective) — plan-reviewer 2차 FAIL/WARN 전면 반영

---

## 서비스 개요

**목적**: 부모가 자신의 목소리로 만든 AI 자장가를 아기에게 들려주는 앱  
**타겟**: 0~3세 자녀를 둔 부모 (부모용 도구, 아동 직접 타겟 아님)  
**핵심 가치**: 녹음 30초면 부모 목소리가 담긴 AI 자장가 생성 — 기존 자장가 앱과 달리 "내 목소리"로 차별화  
**플랫폼**: iOS + Android (React Native + Expo Bare 워크플로)  
**개발 체계**: 1인 개발 + Claude 어시스트, 주말 중심 10~14주 MVP (앱스토어 심사 포함)

## 기능 스펙

### F1 — 회원가입/로그인

**동작 명세:**
1. 앱 첫 실행 시 스플래시 → 온보딩 (개인정보 동의 선행, F13 연계)
2. 이메일+비밀번호 가입 또는 소셜(Apple/Google) 로그인 선택
3. 가입 완료 즉시 7일 Premium 트라이얼 자동 시작 (F14 연계)
4. 이미 계정이 있으면 로그인 → 기존 생성 음원 복원

**유저 시나리오:**
- Happy path: 소셜 로그인 탭 → 동의 → 가입 완료 → 트라이얼 시작 알림
- Edge case: 이메일 중복 가입 → "이미 등록된 이메일" 안내 + 로그인 유도
- Edge case: 소셜 토큰 만료 → 재인증 요청

**수용 기준:**
- Given 앱 첫 실행 / When 소셜 로그인 완료 / Then 7일 트라이얼 상태로 홈 진입
- Given 이메일 가입 / When 이미 등록된 이메일 입력 / Then 에러 메시지 + 로그인 버튼 노출
- Given 로그인 세션 만료 / When 앱 재실행 / Then 로그인 화면 이동 (음원 데이터는 유지)

**우선순위:** Must — 모든 기능의 선행 조건

---

### F2 — 목소리 녹음

**동작 명세:**
1. 자장가 선택(F5) 후 녹음 모드 선택 화면 진입 (허밍 / 쉬 모드)
2. 녹음 가이드 화면 — 모드별 예시 재생 + 텍스트 안내
3. 마이크 권한 요청 (미승인 시 설정 유도)
4. 카운트다운 3초 → 녹음 시작
5. 실시간 음량 파형 표시
6. 30초 최소 / 60초 최대 — 최소 미만이면 자동 연장 안내, 최대 도달 시 자동 종료
7. 녹음 완료 → 미리듣기 → 다시 녹음 / 사용하기 선택
8. "다시 녹음"은 무제한 허용

**라이브 녹음 강제 정책:**
- 파일 업로드 UI 없음
- 녹음 시작 전 challenge-response 문구 표시 ("지금 직접 따라 읽어주세요: [문구]") — 제3자 목소리 업로드 차단

**유저 시나리오:**
- Happy path: 허밍 모드 → 가이드 확인 → 30초 허밍 → 미리듣기 → 사용하기
- Edge case: 마이크 권한 거부 → 팝업으로 설정 이동 유도
- Edge case: 30초 미만 종료 → "조금 더 녹음해주세요" 안내 + 이어서 또는 재시작

**수용 기준:**
- Given 녹음 시작 / When 60초 경과 / Then 자동 종료 + 미리듣기 화면 이동
- Given 마이크 권한 미승인 / When 녹음 버튼 탭 / Then 권한 안내 팝업 노출
- Given 30초 미만 녹음 / When 종료 버튼 탭 / Then 연장 유도 다이얼로그 노출

**우선순위:** Must — F3, F4의 선행 조건

---

### F3 — 샘플 품질 검증

**동작 명세:**
1. 녹음 완료 즉시 클라이언트에서 1차 검증 (길이 / 음량 / 클리핑)
2. 서버로 업로드 후 서버측 2차 검증 (SNR, 배경잡음 분석)
3. 검증 통과 → AI 생성(F4) 진행
4. 검증 실패 → 실패 유형별 안내 메시지 + 재녹음 유도

**검증 기준:**
| 항목 | 기준 |
|---|---|
| 길이 | 30초 이상 |
| 음량 | RMS -40dB ~ -6dB |
| 클리핑 | 피크 3회 이하 |
| SNR (서버) | 15dB 이상 |

**실패 메시지 예시:**
- 너무 조용함: "조금 더 크게 녹음해주세요"
- 잡음 과다: "조용한 공간에서 다시 녹음해주세요"
- 너무 짧음: "30초 이상 녹음이 필요해요"

**수용 기준:**
- Given 녹음 파일 / When 음량 기준 미달 / Then 재녹음 화면 + 해당 이유 안내
- Given 녹음 파일 / When 모든 기준 통과 / Then 자동으로 AI 생성 단계 진행
- Given 서버 검증 실패 / When 응답 수신 / Then 재녹음 유도 (이전 녹음 파일 삭제)

**우선순위:** Must — F4의 직전 게이트

---

### F4 — AI 음원 생성

**동작 명세:**
1. 검증된 목소리 샘플 + 선택한 PD 자장가 곡 → 서버 전송
2. 생성 중 대기 화면 (애니메이션 + 예상 시간 안내, 30~90초)
3. 생성 완료 → mp3 클라이언트에 다운로드 + 로컬 캐시 저장
4. 생성 실패 (타임아웃/서버 오류) → 재시도 버튼 + 오류 메시지

**무료 티어 생성 횟수 제한:**
- 무료 플랜 유저: **계정당 총 3회** AI 음원 생성 허용
- 3회 소진 후 4번째 시도 → S14 업그레이드 팝업 ("Premium 구독 시 무제한 생성")
- 횟수 표시: 자장가 선택(S07), 녹음 모드 선택(S08), 녹음(S10), 업그레이드 팝업(S14)에 "생성 N/3" 표시
- Premium/Trial 유저: 횟수 제한 없음, 횟수 UI 미노출
- 서버사이드 enforcement 필수 (클라이언트 우회 방지)
- 재녹음/재시도는 횟수 차감 없음 — 최종 생성 성공 시에만 1회 차감

**생성 파라미터:**
- 출력 길이: 약 3분 (loop용 단위 클립, 클라이언트에서 seamless loop)
- 포맷: MP3 128kbps 이상
- GPU 인프라: Replicate / Modal / RunPod 중 설계 단계 결정
- 모델: OpenVoice V2 / F5-TTS / RVC / CosyVoice 벤치마크 후 결정 (마일스톤 0)

**데이터 보관:**
- 업로드된 목소리 샘플: 생성 완료 후 서버에서 **24시간 이내** 삭제 (F13 연계)
- 생성된 mp3: S3/R2 저장 + 유저 계정 연결

**백그라운드 생성 완료 재진입 동선:**
- 생성 중 앱 종료 → 백그라운드에서 서버 처리 완료
- 유저가 앱 재진입 시: S06 홈에 **"생성 완료 음원 있음" 카드** 자동 노출 (새 배지 + 탭 시 S13 재생 진입)
- 푸시 알림 없이도 동선 완결 (V1에 푸시 추가 안 함)

**유저 시나리오:**
- Happy path: 업로드 → 30초 대기 → 재생 화면 자동 이동
- Edge case: 90초 초과 → 타임아웃 안내 + 재시도 / **홈으로 이동** 선택
- Edge case: 서버 과부하 → 큐 대기 안내 ("잠시 기다려주세요, N명이 기다리고 있어요")
- Edge case: 생성 중 앱 종료 후 재진입 → S06 홈에 "생성 완료 음원 있음" 카드 노출
- Edge case: 무료 유저 3/3 소진 → S14 업그레이드 팝업 (횟수 소진 안내)

**수용 기준:**
- Given 샘플 업로드 완료 / When 생성 성공 / Then 90초 이내 mp3 반환 + 재생 화면 이동
- Given 생성 실패 / When 90초 초과 / Then 재시도 버튼 + "홈으로 이동" 버튼 노출
- Given 생성 완료 / When 유저가 앱을 닫고 재진입 / Then S06 홈에 "생성 완료 음원 있음" 카드 노출
- Given 무료 유저 생성 횟수 3회 소진 / When 생성 시도 / Then S14 업그레이드 팝업 (횟수 소진 안내 포함)
- Given 무료 유저 생성 횟수 미소진 / When S07~S10 진입 / Then "생성 N/3" 횟수 표시
- Given 생성 성공 / When 서버 확인 / Then 해당 계정 생성 횟수 1 증가 (서버사이드 카운터)
- Given 재시도 / When 동일 샘플 재생성 / Then 횟수 차감 없이 진행 (이미 시도한 건)
- Given 무료 유저 3회 소진 / When S14 팝업에서 "지금은 괜찮아요" 탭 / Then S11 미리듣기 복귀 + "자장가 만들기" 버튼 비활성 + "무료 3회 소진 — 구독하면 무제한" 가이드 + "구독하기" CTA (S15 연결)

**우선순위:** Must — F5와 함께 핵심 가치 전달

### F5 — PD 자장가 6곡 선택

**동작 명세:**
1. 녹음 진입 전 자장가 선택 화면
2. 6곡 목록 표시 (곡명 + 30초 미리듣기 버튼)
3. 1곡 선택 → 녹음 화면(F2)으로 이동

**수록 곡 목록 (전곡 Public Domain 확정):**
1. 브람스 자장가 (Brahms' Lullaby)
2. 모차르트 자장가 (Mozart's Lullaby)
3. 슈베르트 자장가 (Schubert's Lullaby)
4. Twinkle Twinkle Little Star
5. Rock-a-bye Baby
6. Hush Little Baby

**참조 멜로디 소스 정책:**
- AI 생성에 사용하는 참조 멜로디는 **직접 MIDI 생성 또는 IMSLP·Musopen 등 CC0 소스만 사용**
- 기존 상업 녹음본(음반) 일체 사용 금지 — PD 악곡이라도 특정 녹음본에 저작인접권 잔존 가능
- AI 생성 방식: 유저 목소리를 MIDI/CC0 소스 위에 입히는 방식으로 구현

**수용 기준:**
- Given 자장가 선택 화면 / When 미리듣기 버튼 탭 / Then 해당 곡 30초 재생 (CC0 소스)
- Given 곡 선택 완료 / When 다음 버튼 탭 / Then 녹음 모드 선택 화면 이동
- Given 기존에 생성한 음원 있음 / When 다른 곡 선택 / Then 새 녹음 필요 안내

**우선순위:** Must — F4의 입력 파라미터

---

### F6 — 재생

**동작 명세:**
1. 생성된 mp3 재생 화면 진입
2. 재생/일시정지, 볼륨 조절 컨트롤
3. Seamless loop: 트랙 끝 도달 직전 crossfade로 자연스럽게 반복
4. 최대 10시간 연속 재생 지원 (클라이언트 loop, 서버 재요청 없음)
5. 타이머 설정 시 해당 시간 후 자동 종료 (F8 연계)

**수용 기준:**
- Given 재생 중 / When 트랙 끝 도달 / Then 무음 없이 루프 재생 (crossfade 300ms 이상)
- Given 10시간 경과 / When 타이머 미설정 / Then 재생 자동 종료 + 알림
- Given 볼륨 조절 / When 슬라이더 이동 / Then 즉시 반영

**우선순위:** Must — 핵심 사용 가치

### F7 — 백그라운드 재생

**동작 명세:**
1. 재생 중 홈 버튼 / 화면 잠금 → 재생 계속 유지
2. 무료 플랜: 백그라운드 재생 불가 (포그라운드 전용) — 단, 7일 트라이얼 기간 중 무제한
3. Premium / Rewarded Ad 언락: 백그라운드 재생 허용
4. 무료 유저가 홈 버튼 → 재생 일시정지 + 백그라운드 업그레이드 유도 팝업

**S13 뒤로 탭 행동:**
- Premium/Trial 유저: 뒤로 탭 → 재생 유지 + S06 홈 이동 + **C06 미니 플레이어 표시**
- 무료 유저: 뒤로 탭 → "재생을 중단할까요?" 팝업 → 확인 시 재생 중단 + S06 이동

**수용 기준:**
- Given Premium 유저 / When 화면 잠금 / Then 재생 계속 + lockscreen 컨트롤 노출
- Given 무료 유저 (트라이얼 아님) / When 화면 잠금 / Then 재생 일시정지 + 업그레이드 팝업
- Given Rewarded Ad 시청 완료 / When 화면 잠금 / Then 오늘 밤 한정 백그라운드 재생 허용
- Given Premium/Trial 유저 / When S13에서 뒤로 탭 / Then 재생 유지 + S06 이동 + C06 미니 플레이어 표시
- Given 무료 유저 / When S13에서 뒤로 탭 / Then "재생을 중단할까요?" 팝업 노출

**우선순위:** Must — 핵심 사용 시나리오 (아기가 잠들면 폰 놓음)

### F8 — 타이머

**동작 명세:**
1. 재생 화면에서 타이머 아이콘 탭
2. 옵션 선택: 30분 / 1시간 / 2시간 / 6시간 / 10시간
3. 타이머 설정 후 화면에 잔여 시간 표시
4. 타임아웃 1분 전 알림 (선택)
5. 타임아웃 → 페이드아웃 후 재생 종료

**수용 기준:**
- Given 타이머 2시간 설정 / When 2시간 경과 / Then 10초 페이드아웃 후 종료
- Given 타이머 설정 중 / When 재생 종료 / Then 타이머 초기화
- Given 타임아웃 1분 전 / When 알림 설정 ON / Then 로컬 푸시 알림 발송
- Given 알림 권한 거부 / When 타임아웃 1분 전 / Then 푸시 미발송 + 앱 내 화면 배너로 대체 안내

- Given 광고 로드 실패 / When 재생 화면 / Then 광고 영역 collapse (UI 깨짐 없음)

### F11 — Rewarded Ad (오늘 밤 백그라운드 언락)

**동작 명세:**

2. 팝업에 "광고 보고 오늘 밤 무료로 쓰기" 버튼 노출
3. Rewarded Ad 시청 완료 → 자정까지 백그라운드 재생 허용
4. **월 7회 제한** (캘린더 월 기준 리셋) — 하루 1회 아님
5. 트라이얼 7일 기간 중에는 Rewarded Ad 버튼 미노출 (이미 Premium 상태)

**수용 기준:**
- Given 광고 시청 완료 / When 화면 잠금 / Then 자정까지 백그라운드 재생 허용
- Given 당월 이미 7회 시청 완료 / When 다시 시도 / Then "이번 달은 이미 모두 사용했어요" 안내
- Given 광고 로드 실패 / When Rewarded Ad 버튼 탭 / Then "광고를 불러오지 못했어요" 안내
- Given 트라이얼 유저 / When 업그레이드 팝업 진입 / Then Rewarded Ad 버튼 미노출 (구독 CTA만)
- Given 캘린더 월 변경 / When 앱 실행 / Then 월 사용 횟수 초기화 (7회 복원)

### F12 — IAP 구독

**동작 명세:**

3. 구독 복원 버튼 (기기 변경 시)
4. 구독 취소: 앱스토어/플레이스토어에서 처리 (앱 내 링크 제공)

**수용 기준:**
- Given 구독 결제 완료 / When 앱 재실행 / Then Premium entitlement 유지
- Given 기기 변경 / When 복원 버튼 탭 / Then 구독 상태 복원
- Given 구독 취소 후 만료일 이전 / When 앱 실행 / Then 만료일까지 Premium 유지

**우선순위:** Must — 주 수익원 (LTD 없음, 구독 전용)

### F13 — 개인정보 동의 + 데이터 삭제

**동작 명세:**

3. 미동의 → 앱 사용 불가 (종료 안내)
4. 설정 화면에서 언제든지 데이터 삭제 요청 가능
   - 목소리 샘플: 서버에서 즉시 삭제
   - 생성된 음원: 유저 선택 (유지 or 삭제)
   - 계정 탈퇴: 모든 데이터 삭제

**법적 근거:**
- 음성 = 생체정보 → 개인정보보호법 / GDPR Art.9 적용
- 제3자 목소리 업로드 금지 TOS 명시

**수용 기준:**
- Given 앱 최초 실행 / When 동의 화면 / Then 동의 없이는 가입 화면 진입 불가
- Given 데이터 삭제 요청 / When 확인 탭 / Then **24시간 이내** 서버 삭제 완료 + 완료 알림
- Given 계정 탈퇴 / When 탈퇴 완료 / Then 로그인 화면 이동 + 로컬 캐시 삭제

**우선순위:** Must — 법적 필수 요건

### F14 — 7일 무료 Premium 체험

**동작 명세:**
1. 신규 가입 완료 즉시 RevenueCat 트라이얼 자동 시작
2. 7일간: 광고 없음 + 백그라운드 무제한 + 오프라인 다운로드 + AI 생성 횟수 제한 없음
3. 트라이얼 D-1: 만료 예정 푸시 알림
4. 만료 → 결제 없으면 무료 플랜 자동 다운그레이드 (배너광고 + 포그라운드만 + AI 생성 3회 한도 리셋)
5. 구현: RevenueCat 트라이얼 설정 토글 (별도 서버 로직 최소화)

**수용 기준:**
- Given 신규 가입 / When 가입 완료 / Then 7일 트라이얼 상태 + 화면 내 트라이얼 배지 노출
- Given 트라이얼 D-1 / When 알림 허용 / Then "내일 트라이얼이 만료돼요" 푸시 발송
- Given 알림 권한 거부 / When 트라이얼 D-1 / Then 푸시 미발송 + 앱 열 때 인앱 배너로 만료 예정 안내
- Given 트라이얼 만료 / When 결제 미완료 / Then 무료 플랜 전환 + 구독 유도 화면 노출

**우선순위:** Must — 전환율 핵심 레버

# Epic 03 — AI 음원 생성

**포함 기능:** F4 (AI 음원 생성), F5 (PD 자장가 선택은 Epic 02에 포함)  
**선행 조건:** Epic 02 완료 (검증된 녹음 샘플 준비)  
**완료 기준:** 서버에서 생성된 mp3를 클라이언트가 수신해 재생 화면 진입 가능

---

## Story 1 — 생성 중 대기 화면

**As a** 유저  
**I want** AI 생성이 진행 중임을 시각적으로 확인하고 싶다  
**So that** 앱이 멈춘 건지 처리 중인지 알 수 있다

### 태스크 체크리스트

- [ ] 생성 중 애니메이션 UI (파형 또는 수면 테마)
- [ ] 예상 대기 시간 표시 ("보통 30~90초 정도 걸려요")
- [ ] 경과 시간 카운터 표시
- [ ] 백그라운드로 이동해도 생성 계속 진행 (포그라운드 복귀 시 진행 상태 복원)
- [ ] 생성 완료 수신 → 재생 화면 자동 이동 (성공 애니메이션)
- [ ] 취소 버튼 (확인 다이얼로그: "취소하면 처음부터 다시 해야 해요")

### 수용 기준

- Given 생성 요청 완료 / When 대기 화면 / Then 애니메이션 + 예상 시간 안내 노출
- Given 생성 성공 / When 완료 수신 / Then 재생 화면 자동 이동
- Given 앱 백그라운드 이동 / When 생성 중 / Then 백그라운드에서 계속 처리
- Given 포그라운드 복귀 / When 생성 완료 / Then 재생 화면 이동 또는 홈에 완료 알림

---

## Story 2 — AI 생성 API 연동

**As a** 시스템  
**I want** 목소리 샘플과 선택 곡을 서버에 전송하고 mp3를 받고 싶다  
**So that** 유저에게 개인화된 AI 자장가를 제공할 수 있다

### 태스크 체크리스트

- [ ] 클라이언트 → 서버: 목소리 샘플(wav/m4a) + 선택 곡 ID 업로드
- [ ] 서버: GPU 추론 잡 큐 등록 + 잡 ID 반환
- [ ] 폴링 또는 웹소켓으로 생성 상태 확인 (pending / processing / done / failed)
- [ ] 완료 시 presigned URL 또는 직접 mp3 반환
- [ ] 클라이언트: mp3 다운로드 + 로컬 캐시 저장
- [ ] 생성된 음원 메타 (곡명, 생성일, 파일 경로) DB 저장

### 수용 기준

- Given 샘플 업로드 완료 / When 서버 처리 / Then 90초 이내 mp3 반환
- Given 90초 초과 / When 타임아웃 / Then 실패 상태 반환 + 클라이언트 재시도 안내
- Given 생성 완료 / When 앱 재실행 / Then 로컬 캐시에서 즉시 재생 가능

---

## Story 3 — 생성 실패 처리

**As a** 유저  
**I want** 생성이 실패했을 때 명확한 안내와 재시도 방법을 알고 싶다  
**So that** 당황하지 않고 다시 시도할 수 있다

### 태스크 체크리스트

- [ ] 타임아웃(90초) 시 실패 화면 노출
- [ ] 서버 오류(5xx) 시 실패 화면 노출
- [ ] 실패 화면: 이유 안내 + 재시도 버튼 + **"홈으로 이동" 버튼** (푸시 알림/알림 예약 없음)
- [ ] "홈으로 이동": 홈(S06) 이동 후 생성 완료 시 **"생성 완료 음원 있음" 카드** 자동 노출 (배지 + 탭 시 S13)
- [ ] 서버 과부하 시 큐 대기 안내 ("잠시 기다려주세요, N명이 기다리고 있어요")
- [ ] 재시도: 동일 샘플 재사용 (재녹음 없이, 횟수 차감 없음)

### 수용 기준

- Given 90초 초과 / When 타임아웃 / Then 실패 화면 + 재시도 버튼 + "홈으로 이동" 버튼
- Given 서버 5xx 오류 / When 응답 / Then "서버에 문제가 생겼어요. 잠시 후 다시 시도해주세요"
- Given 재시도 버튼 탭 / When / Then 동일 샘플로 재생성 요청 (재녹음 없이, 생성 횟수 차감 없음)
- Given "홈으로 이동" 탭 / When / Then S06 홈 이동 (푸시 알림 없음)
- Given 백그라운드에서 생성 완료 / When 유저 홈 진입 / Then "생성 완료 음원 있음" 카드 자동 노출

## Story 4 — 목소리 샘플 서버 자동 삭제

**As a** 시스템  
**I want** 생성 완료 후 업로드된 목소리 샘플을 즉시 삭제하고 싶다  
**So that** 생체정보를 최소 기간만 보관하고 법적 요건을 충족할 수 있다

### 태스크 체크리스트

- [ ] 생성 완료 즉시 서버에서 원본 샘플 파일 삭제
- [ ] 생성 실패 / 취소 시에도 업로드 파일 삭제 (24시간 TTL 보장)
- [ ] 삭제 완료 로그 기록 (감사 목적)
- [ ] 클라이언트 로컬 임시 녹음 파일도 생성 완료 후 삭제

### 수용 기준

- Given 생성 완료 / When 완료 이벤트 / Then 서버 샘플 파일 즉시 삭제
- Given 생성 실패 / When 24시간 경과 / Then 미삭제 파일 자동 TTL 삭제
- Given 삭제 완료 / When / Then 삭제 로그 저장 (파일 ID + 타임스탬프)

---

## Story 5 — 홈 화면 음원 목록

**As a** 유저  
**I want** 내가 생성한 음원 목록을 홈에서 확인하고 싶다  
**So that** 이전에 만든 자장가를 바로 재생할 수 있다

### 태스크 체크리스트

- [ ] 홈 화면: 생성된 음원 카드 목록 (곡명 + 생성일 + 재생 버튼)
- [ ] 생성된 음원 없을 시: 빈 상태 UI + "자장가 만들기" CTA
- [ ] 음원 카드 탭 → 재생 화면 이동
- [ ] 음원 삭제 (스와이프 또는 롱탭 → 삭제 확인)
- [ ] 최대 음원 저장 개수 정책 (V1: 무제한 또는 3개 제한 — 설계 단계 결정)

### 수용 기준

- Given 음원 생성 완료 / When 홈 진입 / Then 목록에 신규 음원 카드 노출
- Given 음원 없음 / When 홈 진입 / Then 빈 상태 + "자장가 만들기" CTA
- Given 카드 탭 / When / Then 해당 음원 재생 화면 이동

## Story 6 — AI 생성 횟수 카운터 (무료 티어 3회 제한)

**As a** 시스템  
**I want** 무료 유저의 AI 음원 생성 횟수를 계정 단위로 제한하고 싶다  
**So that** 구독 전환 압력을 형성하고 서버 비용을 통제할 수 있다

### 태스크 체크리스트

- [ ] 서버: 계정별 생성 횟수 카운터 저장 (DB — 무료 유저 총 3회 한도)
- [ ] 서버: 생성 성공 시 카운터 +1 (재시도/재녹음은 차감 없음, 최종 성공 시에만 카운트)
- [ ] 서버: 생성 요청 시 카운터 초과 여부 사전 검증 (클라이언트 우회 방지)
- [ ] 클라이언트: S07(자장가 선택) / S08(녹음 모드 선택) / S10(녹음) 화면에 "생성 N/3" 표시 (무료 유저만)
- [ ] 클라이언트: 3/3 소진 후 생성 시도 → S14 업그레이드 팝업 진입 (횟수 소진 안내 + 구독 CTA)
- [ ] Premium/Trial 유저: 횟수 카운터 적용 안 함, 횟수 UI 미노출
- [ ] 트라이얼 만료 후 무료 다운그레이드 시: 카운터 리셋 없음 (기존 소진 횟수 유지)

### 수용 기준

- Given 무료 유저 / When S07 진입 / Then "생성 N/3" 잔여 횟수 표시
- Given 무료 유저 생성 3회 소진 / When 4번째 생성 시도 / Then S14 업그레이드 팝업 (횟수 소진 메시지 포함)
- Given 생성 실패 후 재시도 / When 동일 세션 재생성 / Then 카운터 추가 차감 없음
- Given 서버에서 횟수 초과 감지 / When 생성 API 요청 / Then 403 응답 + 클라이언트 팝업 진입
- Given Premium 유저 / When S07 진입 / Then 횟수 표시 없음, 생성 무제한

## 목표

부모가 자신의 목소리로 만든 자장가를 아기에게 들려주는 앱. AI clone 이 아니라 진짜 부모가 직접 부른 자장가 — 녹음 1 loop 만으로 부모 목소리가 담긴 자장가 완성. 기존 자장가 앱과 달리 "내 목소리"로 차별화.

## 사용자

- **타겟**: 0~3세 자녀를 둔 부모 (부모용 도구, 아동 직접 타겟 아님)
- **플랫폼**: iOS + Android (React Native + Expo Bare 워크플로)
- **개발 체계**: 1인 개발 + Claude 어시스트, 주말 중심 10~14주 MVP (앱스토어 심사 포함)

## 기능 스펙

### F1 — 회원가입/로그인

**동작 명세:**
1. 앱 첫 실행 시 스플래시 → 온보딩 (개인정보 동의 선행, F13 연계)
2. 이메일+비밀번호 가입 또는 소셜(Apple/Google) 로그인 선택
3. 가입 완료 즉시 7일 Premium 트라이얼 자동 시작 (F14 연계 — server-side custom trial)
4. 이미 계정이 있으면 로그인 → 기존 생성 음원 복원

**유저 시나리오:**
- Happy path: 소셜 로그인 탭 → 동의 → 가입 완료 → 트라이얼 시작 알림
- Edge case: 이메일 중복 가입 → "이미 등록된 이메일" 안내 + 로그인 유도
- Edge case: 소셜 토큰 만료 → 재인증 요청

**수용 기준:**
- Given 앱 첫 실행 / When 소셜 로그인 완료 / Then 서버가 trial_expires_at = NOW()+7d 설정 → 7일 트라이얼 상태로 홈 진입
- Given 이메일 가입 / When 이미 등록된 이메일 입력 / Then 에러 메시지 + 로그인 버튼 노출
- Given 로그인 세션 만료 / When 앱 재실행 / Then 로그인 화면 이동 (음원 데이터는 유지)

**우선순위:** Must — 모든 기능의 선행 조건

---

### F2 — 목소리 녹음

**동작 명세:**
1. 자장가 선택(F5) 후 녹음 가이드 화면 진입
2. 녹음 가이드 화면 — 선택한 곡 예시 재생 + 텍스트 안내 + 한국어 가사 미리보기 박스 + 헤드폰 권장 안내. 가이드 문구: "1 loop 동안 자유롭게 — 따라불러도, 허밍해도, 쉬쉬 소리만 내도 좋습니다. 더 많이 녹음할수록 더 풍성해집니다."
3. 마이크 권한 요청 (미승인 시 설정 유도)
4. 이어폰 미착용 감지 시 1회 경고 팝업 노출 (단, "그래도 진행" 선택 시 경고 없이 진행)
5. 카운트다운 3초 → 녹음 시작
6. 카운트다운 종료와 동시에 선택된 자장가 BGM이 30% 볼륨으로 재생되며, 화면에는 곡별 한국어 가사(1절 4~6줄)가 표시됨. 녹음 종료 시 BGM도 즉시 정지.
7. 실시간 음량 파형 표시
8. 녹음 길이는 선택한 곡의 "1 loop" (곡 전체 1회 재생 길이) 기준 — 1 loop 종료 시 자동 녹음 종료
9. 녹음 완료 → 미리듣기 → 다시 녹음 / 사용하기 선택
10. "다시 녹음"은 무제한 허용. 추가 녹음 유도 문구 노출 ("한 번 더 녹음하면 더 풍성해집니다")
11. 재생 셔플 시 직전 재생한 녹음 클립은 제외 (F4 연계, N≥2인 경우에만 적용 — N=1이면 단순 반복)

**라이브 녹음 강제 정책:**
- 파일 업로드 UI 없음
- 라이브 녹음 강제 + TOS 명시로 제3자 목소리 업로드 차단
- 가사 박스는 안티봇 검증 수단이 아님 — 이번 마일스톤은 검증 약화 risk 수용, 후속 마일스톤에서 보완 검토

**유저 시나리오:**
- Happy path: 가이드(헤드폰 권장 + 가사 미리보기) → 이어폰 착용 확인 → 카운트다운 → BGM 30% + 가사 박스 노출 상태에서 1 loop 자유 녹음 → 자동 종료 → 미리듣기 → 사용하기
- Edge case: 마이크 권한 거부 → 팝업으로 설정 이동 유도
- Edge case: 이어폰 미착용 → 1회 경고 팝업 → "그래도 진행" 시 경고 없이 진행
- Edge case: BGM 음원 로드 실패 → BGM 없이 녹음 진행 + "음악 없이 녹음할게요" 토스트
- Edge case: 가사 미준비 곡(fallback) → 가사 박스 숨김 + "자유롭게 따라불러 주세요" 안내. 출시 블로킹 안 함 (6곡 모두 가사 준비로 결정 — 안전망 케이스)
- Edge case: 다시 녹음 시 BGM은 같은 곡을 처음부터 재생

**수용 기준:**
- Given 녹음 시작 / When 1 loop 종료 / Then 자동 종료 + 미리듣기 화면 이동
- Given 마이크 권한 미승인 / When 녹음 버튼 탭 / Then 권한 안내 팝업 노출
- Given 이어폰 미착용 / When 녹음 시작 시도 / Then 경고 팝업 1회 노출 ("그래도 진행" 버튼 포함)
- Given "그래도 진행" 선택 / When 이후 녹음 재시도 / Then 이어폰 경고 미노출 (1회만)
- Given 카운트다운 종료 / When 녹음 시작 / Then BGM 30% 볼륨 재생 시작 + 가사 박스 노출
- Given 녹음 진행 중 / When 1 loop 종료(수동/자동) / Then BGM 즉시 정지
- Given BGM 음원 로드 실패 / When 카운트다운 종료 / Then BGM 없이 진행 + 토스트 안내
- Given 곡 가사 미준비 / When 녹음 가이드 진입 / Then 가사 박스 숨김 + "자유롭게 따라불러 주세요" 안내
- Given 저장된 클립 N=1 / When 재생 루프 / Then 셔플 분기 미적용, 동일 클립 단순 반복
- Given 저장된 클립 N≥2 / When 재생 루프 / Then 직전 클립 제외 후 나머지 N-1개 풀에서 다음 클립 선택

**우선순위:** Must — F3, F4의 선행 조건

### F3 — 샘플 품질 검증

**동작 명세:**
1. 녹음 완료 즉시 클라이언트에서 1차 검증 (길이 / 음량 / 클리핑)
2. 서버로 업로드 후 서버측 2차 검증 (SNR, 배경잡음 분석)
3. 검증 통과 → 음원 후처리 생성(F4) 진행
4. 검증 실패 → 실패 유형별 안내 메시지 + 재녹음 유도

**검증 기준:**
| 항목 | 기준 |
|---|---|
| 길이 | 1 loop 이상 (곡 전체 1회 재생 길이) |
| 음량 | RMS -40dB ~ -6dB |
| 클리핑 | 피크 3회 이하 |
| SNR (서버) | 15dB 이상 |

**실패 메시지 예시:**
- 너무 조용함: "조금 더 크게 녹음해주세요"
- 잡음 과다: "조용한 공간에서 다시 녹음해주세요"
- 너무 짧음: "1 loop 전체를 녹음해주세요"

**수용 기준:**
- Given 녹음 파일 / When 음량 기준 미달 / Then 재녹음 화면 + 해당 이유 안내
- Given 녹음 파일 / When 모든 기준 통과 / Then 자동으로 음원 후처리 생성 단계 진행
- Given 서버 검증 실패 / When 응답 수신 / Then 재녹음 유도 (이전 녹음 파일 삭제)

**우선순위:** Must — F4의 직전 게이트

---

### F4 — 음원 후처리 생성

**동작 명세:**
1. 검증된 목소리 샘플 + 선택한 PD 자장가 곡 → 서버 전송
2. 생성 중 대기 화면 (애니메이션 + 예상 시간 안내, 30초 이내)
3. 생성 완료 → mp3 클라이언트에 다운로드 + 로컬 캐시 저장
4. 생성 실패 (타임아웃/서버 오류) → 재시도 버튼 + 오류 메시지

**DSP 파이프라인 (서버, ffmpeg DSP pipeline 기반 — librosa는 분석/검증용 한정):**

| 단계 | 처리 | 도구 |
|---|---|---|
| 노이즈 제거 | afftdn 필터 | ffmpeg |
| EQ | equalizer 필터 | ffmpeg |
| Reverb | aecho 필터 | ffmpeg |
| Loop crossfade | acrossfade 필터 | ffmpeg |
| 클립 concat | N개 클립 직전 제외 셔플 concat | ffmpeg |
| SNR 측정/검증 | SNR 15dB 이상 확인, 음량·클리핑 검출 | librosa (분석 전용) |

**N=1 / N≥2 클립 수 분기:**
- **N=1 (클립 1개)**: DSP 후처리(noize/EQ/reverb/crossfade) 동일 적용. 셔플 분기 미적용 — 단순 loop 반복. "더 녹음할수록 풍성해짐" 문구는 셔플 다양성 확보를 의미하며, N=1에서도 UI에 자연스럽게 표시됨.
- **N≥2 (클립 2개 이상)**: DSP 후처리 + 직전 클립 제외 셔플 concat. 셔플 풀 = N-1 (직전 제외 후 나머지).

**무료 티어 생성 횟수 제한:**
- 무료 플랜 유저: **계정당 총 3회** 후처리 음원 생성 허용
- 3회 소진 후 4번째 시도 → S14 업그레이드 팝업 ("Premium 구독 시 무제한 생성")
- 횟수 표시: 자장가 선택(S07), 녹음 가이드(S09), 녹음(S10), 업그레이드 팝업(S14)에 "생성 N/3" 표시
- Premium/Trial 유저: 횟수 제한 없음, 횟수 UI 미노출
- 서버사이드 enforcement 필수 (클라이언트 우회 방지)
- 재녹음/재시도는 횟수 차감 없음 — 최종 생성 성공 시에만 1회 차감

**생성 파라미터:**
- 출력 길이: 약 3분 (loop용 단위 클립, 클라이언트에서 seamless loop)
- 포맷: MP3 128kbps 이상
- 셔플: 직전 재생한 클립 제외 후 다음 클립 선택 (N≥2인 경우에만 적용)

**데이터 보관:**
- 업로드된 목소리 샘플: 생성 완료 후 서버에서 **24시간 이내** 삭제 (F13 연계)
- 생성된 mp3: S3/R2 저장 + 유저 계정 연결

**가이드 자산:**
- 녹음 가이드 BGM 및 30초 미리듣기: `music/*` (CC0 소스)
- 곡별 미리듣기 샘플: `apps/api/static/previews/*`

**백그라운드 생성 완료 재진입 동선:**
- 생성 중 앱 종료 → 백그라운드에서 서버 처리 완료
- 유저가 앱 재진입 시: S06 홈에 **"생성 완료 음원 있음" 카드** 자동 노출 (새 배지 + 탭 시 S13 재생 진입)
- 푸시 알림 없이도 동선 완결 (V1에 푸시 추가 안 함)

**유저 시나리오:**
- Happy path: 업로드 → 30초 이내 대기 → 재생 화면 자동 이동
- Edge case: 30초 초과 → 타임아웃 안내 + 재시도 / **홈으로 이동** 선택
- Edge case: 서버 과부하 → 큐 대기 안내 ("잠시 기다려주세요, N명이 기다리고 있어요")
- Edge case: 생성 중 앱 종료 후 재진입 → S06 홈에 "생성 완료 음원 있음" 카드 노출
- Edge case: 무료 유저 3/3 소진 → S14 업그레이드 팝업 (횟수 소진 안내)

**수용 기준:**
- Given 샘플 업로드 완료 / When 생성 성공 / Then 30초 이내 mp3 반환 + 재생 화면 이동
- Given 생성 실패 / When 30초 초과 / Then 재시도 버튼 + "홈으로 이동" 버튼 노출
- Given 생성 완료 / When 유저가 앱을 닫고 재진입 / Then S06 홈에 "생성 완료 음원 있음" 카드 노출
- Given 무료 유저 생성 횟수 3회 소진 / When 생성 시도 / Then S14 업그레이드 팝업 (횟수 소진 안내 포함)
- Given 무료 유저 생성 횟수 미소진 / When S07·S09·S10 진입 / Then "생성 N/3" 횟수 표시
- Given 생성 성공 / When 서버 확인 / Then 해당 계정 생성 횟수 1 증가 (서버사이드 카운터)
- Given 재시도 / When 동일 샘플 재생성 / Then 횟수 차감 없이 진행 (이미 시도한 건)
- Given 무료 유저 3회 소진 / When S14 팝업에서 "지금은 괜찮아요" 탭 / Then S11 미리듣기 복귀 + "자장가 만들기" 버튼 비활성 + "무료 3회 소진 — 구독하면 무제한" 가이드 + "구독하기" CTA (S15 연결)
- Given DSP 처리 완료 / When 서버 검증 / Then 단조로움·이음새·노이즈 3항목 자동 체크 통과 (M0 DSP self-test 기준 충족)
- Given 클립 N=1 / When DSP 파이프라인 실행 / Then 셔플 concat 미적용 + DSP 후처리(노이즈/EQ/reverb/crossfade) 정상 적용
- Given 클립 N≥2 / When DSP 파이프라인 실행 / Then 직전 클립 제외 후 N-1개 풀에서 셔플 concat 적용

4. RevenueCat 역할 범위: Premium 구독 결제 (`purchasePackage`) 만 담당. Trial 관리 일체 담당 X
5. 트라이얼 D-1: 만료 예정 푸시 알림 (서버 `trial_expires_at` 기준)
6. 만료 → 결제 없으면 무료 플랜 자동 다운그레이드 (배너광고 + 포그라운드만)
7. 만료 후 무료 다운그레이드: Trial 기간 중 생성한 횟수는 `generation_counter` 에 가산 X. Trial 진입 전 소진 횟수 유지
8. Trial 전환율 측정: RevenueCat 분석 데이터 미사용 → 자체 DB (`trial_expires_at` + `subscriptions.created_at`) 기준 측정

**수용 기준:**
- Given 신규 가입 / When 가입 완료 / Then 서버 `trial_expires_at` = 가입 시각 + 7일 설정 + 화면 내 트라이얼 배지 노출
- Given 앱 실행 / When `trial_expires_at > NOW()` / Then `is_premium = true` (RevenueCat 구독 없어도)
- Given 트라이얼 D-1 / When 알림 허용 / Then "내일 트라이얼이 만료돼요" 푸시 발송
- Given 알림 권한 거부 / When 트라이얼 D-1 / Then 푸시 미발송 + 앱 열 때 인앱 배너로 만료 예정 안내
- Given 트라이얼 만료 / When `trial_expires_at <= NOW()` AND RevenueCat 구독 없음 / Then 무료 플랜 전환 + S15 구독 유도 화면 노출
- Given Trial 기간 중 DSP 생성 성공 / When 카운터 조회 / Then `generation_counter` 변경 없음 (Premium 동등 취급)
- Given Trial 만료 후 무료 다운그레이드 / When 이미 Trial 전 2회 소진 상태 / Then `generation_counter = 2` 유지 (Trial 중 생성분 무가산)

**우선순위:** Must — 전환율 핵심 레버

| **무료 3회 제한 이탈 리스크** | 3회 소진 유저 전환율 vs 이탈율 모니터링. 필요 시 한도 완화 실험 가능 |
| 앱스토어 심사 거절 | 재제출 2주 버퍼 산입, IAP 가이드라인 사전 준수 검토 |

## 시장 맥락

**경쟁/유사 서비스:** Lullaby Generator 류 앱 (부모 목소리 개인화 없음), 일반 자장가 앱
**차별점:** AI clone이 아닌 "진짜 부모가 직접 부른" 자장가 — 아기에게 익숙한 목소리로 수면 유도. DSP 후처리로 볼륨·잡음 정제 + seamless loop 제공.
**배경:** AI 합성 품질 불확실성 + M0 벤치마크 미실행 병목 → "부모가 직접 부른 녹음" 가치로 피벗. 초기 개인화 자장가 앱 공백 시장 진입.
**인프라 비용:** GPU warm pool 불필요 → ffmpeg CPU 워커 1대로 MVP 시작. 수요 증가 시 워커 수평 확장.
**경쟁 리서치:** 마일스톤 0 병행 수행 — 앱스토어 검색어 "lullaby", "baby sleep", "baby voice" 상위 20개 앱 BM·리뷰 스캔 → 카테고리 공백 검증 → 경쟁사 pain point 3가지 추출 → V1 포지셔닝 reconfirm
